        • static java.util.Iterator<com.atlassian.jira.issue.Issue> query(java.lang.String jql)

          Executes the jql and returns matching issues. You can iterate over the issues in the response, for example to edit or transition each one: 

           Jql.query('project = FOO').each {
     // do something with it
 }
          Or filter them further: 
           Jql.query('project = FOO').findAll {
     // condition
 }
          If you want to run the query overriding permissions and/or without query validation, use: 
           Jql.query('project = FOO') {
     skipValidation = true
     overrideSecurity = true
 }
          If you only need the top N: 
           Jql.query('project = FOO').take(10)
          DO NOT attempt to convert a potentially unlimited set to a List, e.g.: 
           Jql.query('project = FOO').toList()
          as this could potentially consume a large amount of memory. If you need a list then limit it to the top N: 
           Jql.query('project = FOO').take(100).toList()
          throws:
          com.atlassian.jira.jql.parser.JqlParseException if the provided JQL could not be parsed
          throws:
          com.adaptavist.atless.jira.issues.exceptions.JqlValidationException if the provided JQL could not be validated
          Parameters:
          jql - A jql query
          specification - configure aspects of the query execution
          Returns:
          an Iterable of Issue objects
